GoSheets — Near-Term Todos
==========================

Stage 2 — Inline Editing (next major milestone)
-------------------------------------------------

[ ] 2-A  Make all grid cells editable with appropriate controls
         - LineEdit  → String
         - SpinBox   → int / float (PROPERTY_HINT_RANGE: use min/max/step from hint_string)
         - CheckBox  → bool
         - OptionButton → PROPERTY_HINT_ENUM (parse hint_string entries)
         - ColorPickerButton → Color
         - Resource picker button → PROPERTY_HINT_RESOURCE_TYPE (EditorInterface resource picker)
         Priority order: String, bool, int/float first; enum + resource ref second.

[ ] 2-B  Sync grid edits ↔ Godot Inspector / resource side panel
         When a cell is changed in GoSheets, the Inspector should reflect it immediately
         (call EditorInterface.edit_resource() or emit changed signal).
         When the Inspector modifies the same resource, GoSheets should refresh the row
         on the next filesystem_changed or via a deferred reload (similar to the existing
         250 ms debounce pattern). Keep them visually consistent.

[ ] 2-C  Preserve Undo/Redo history for all grid edits  ← CRITICAL
         Every cell write must be wrapped in EditorUndoRedoManager:
           undo_redo.create_action("Edit <ResourceType>.<property>")
           undo_redo.add_do_property(resource, property, new_value)
           undo_redo.add_undo_property(resource, property, old_value)
           undo_redo.commit_action()
         Batch multi-cell edits into a single undo action.
         Resource must also be saved (ResourceSaver) as part of the do/undo.
